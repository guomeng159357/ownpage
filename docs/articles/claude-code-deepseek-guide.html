Claude Code 安装与 DeepSeek 接入手册

详细介绍 Claude Code CLI 的安装配置流程，以及如何通过 CC Switch 图形化工具接入 DeepSeek API，附完整的常见问题排查指南。

一、概述

1.1 什么是 Claude Code

Claude Code 是 Anthropic 官方推出的终端 AI 编程助手，它可以直接在命令行中理解你的整个代码库、执行操作、编辑文件、运行命令，并与 GitHub 等工具深度集成。你可以把它理解为一位驻扎在你项目目录里的资深开发者——它能读代码、写代码、调试、甚至帮你管理 Git 工作流。

Claude Code 同时提供以下形态：

形态说明
CLI 命令行终端中直接运行 claude，功能最完整
VS Code 扩展在 IDE 侧边栏中使用
桌面应用Mac/Windows 原生应用（JetBrains 插件也可用）
Web 版claude.ai/code，无需安装

本手册以 CLI 命令行版本 为主要安装目标。

1.2 为什么要接入 DeepSeek

Claude Code 默认对接 Anthropic 官方 API（Claude 系列模型）。但如果你在中国大陆使用，可能会面临以下痛点：

  • 网络访问困难：Anthropic API 在国内可能无法直连
  • 付费门槛：Claude Pro 订阅 $20/月（约 ¥140/月），对于个人开发者不便宜
  • 按量付费更灵活：DeepSeek 提供按 token 计费，日常开发可能不到 ¥20/月

DeepSeek V4 系列模型的能力已经非常出色，而且——关键点——DeepSeek 提供了原生的 Anthropic 兼容 API 端点，这意味着无需任何中间代理即可直接替换。

成本参考：

方案费用
Claude Pro 订阅$20/月（约 ¥140/月）
DeepSeek V4 按量付费¥1-3 / 百万 token，日常使用约 ¥10-30/月
DeepSeek 缓存命中¥0.02 / 百万 token（命中率可达 98%+）

1.3 本手册的技术路线

经过社区大量验证，当前最简洁、最稳定的方案是：

Claude Code CLI  →  CC Switch（配置管理）  →  DeepSeek Anthropic 兼容端点

为什么不需要代理层？过去有人推荐 LiteLLM、one-api 等中间代理来做 API 格式转换，但 DeepSeek 现在已经原生提供 https://api.deepseek.com/anthropic 端点，直接支持 Anthropic 协议。CC Switch 的作用仅仅是帮你图形化管理环境变量配置，不引入额外的网络跳转和故障点。

二、环境准备

2.1 系统要求

操作系统最低版本
macOSmacOS 12 (Monterey) 及以上
WindowsWindows 10 及以上
LinuxUbuntu 20.04+、Debian 11+、CentOS 7+

硬件无特殊要求，Claude Code 本身不运行本地模型，计算在云端完成。

前置依赖总览：

依赖最低版本说明
Node.js18.0+（推荐 20 LTS 或 22 LTS）Claude Code 的运行环境
npm9.0+随 Node.js 安装，用于安装 Claude Code
Git2.38+（最低 2.23）用于仓库分析、变更追踪、提交管理
⚠️ 重要：以上三个依赖缺一不可。尤其是 Git——Claude Code 深度依赖 Git 来理解代码历史、追踪文件变更、生成 commit message 等。Windows 用户必须安装 Git for Windows。

2.2 安装 Node.js 18+

推荐方式：使用 nvm 安装

macOS / Linux：

# 安装 nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash
# 重启终端或执行
source ~/.zshrc
# 安装 Node.js 20 LTS（推荐）
nvm install 20
# 验证版本
node --version   # 应输出 v20.x.x
npm --version    # 应输出 10.x.x

Windows 用户推荐使用 nvm-windows，或从 nodejs.org 下载 LTS 安装包直接安装。

2.3 安装 Git 2.38+

Claude Code 深度依赖 Git。即使你的项目不使用 Git 做版本控制，Claude Code 也需要 Git 来运行。

macOS

# 方式一：Xcode Command Line Tools
xcode-select --install

# 方式二：Homebrew（推荐）
brew install git
git --version   # 应输出 git version 2.47.x 或更高

Windows

下载并安装 Git for Windows，安装时注意：默认编辑器选 VS Code；PATH 环境选 "Git from the command line and also from 3rd-party software"；行结束符选 "Checkout as-is, commit Unix-style"。

⚠️ Windows 特别提醒：Claude Code 在 Windows 上运行时，内部会使用 Git Bash 执行命令。即使你日常使用 PowerShell 或 CMD，也必须安装 Git for Windows。

Linux

# Ubuntu / Debian
sudo apt update && sudo apt install git
# 如果版本过低
sudo add-apt-repository ppa:git-core/ppa && sudo apt update && sudo apt install git

# CentOS / RHEL: sudo yum install git
# Fedora: sudo dnf install git
# Arch: sudo pacman -S git

安装后基础配置（强烈建议）：

git config --global user.name "你的姓名"
git config --global user.email "你的邮箱@example.com"

2.4 注册 DeepSeek 并获取 API Key

  1. 访问 platform.deepseek.com
  2. 注册账号（支持手机号注册）
  3. 登录后，在左侧菜单找到 "API Keys"
  4. 点击"创建新的 API Key"，输入名称（如 claude-code），点击创建
  5. 立即复制 API Key！它只显示一次
  6. 建议在"用量管理"页面充值 ¥10-50 用于测试
⚠️ 安全提示：API Key 相当于你的账户密码，请勿泄露给他人或提交到公开仓库。

2.5 环境准备检查清单

node --version   # 应 ≥ v18.0.0（推荐 v20.x 或 v22.x）
npm --version    # 应 ≥ 9.0.0（推荐 10.x）
git --version    # 应 ≥ 2.38.0（推荐 2.47+）
检查项预期结果如未通过
node --versionv18.0.0 及以上见 2.2 节
npm --version9.0.0 及以上升级 Node.js
git --version2.38.0 及以上见 2.3 节
DeepSeek API Key已创建并复制保存见 2.4 节
DeepSeek 余额≥ ¥10（建议）platform.deepseek.com 充值

三、安装 Claude Code

3.1 通过 npm 全局安装

npm install -g @anthropic-ai/claude-code

国内用户可使用淘宝镜像加速：

npm install -g @anthropic-ai/claude-code --registry=https://registry.npmmirror.com

安装过程通常持续 1-2 分钟。

3.2 验证安装

claude --version   # 应输出版本号如 v1.0.x
claude --help      # 查看帮助信息

3.3 跳过新手引导

Claude Code 首次启动时默认引导登录 Anthropic 账号。由于我们使用 DeepSeek，需要跳过：

macOS / Linux：

echo '{"hasCompletedOnboarding": true}' > ~/.claude.json

Windows (PowerShell)：

echo '{"hasCompletedOnboarding": true}' > $env:USERPROFILE\.claude.json

也可以手动创建文件 ~/.claude.json，内容为：

{
"hasCompletedOnboarding": true
}

四、安装并配置 CC Switch

4.1 CC Switch 简介

CC Switch 是一款跨平台桌面应用，专为 Claude Code、Codex、OpenCode、Gemini CLI 等 AI 编程工具提供可视化的供应商配置管理。

功能说明
一键切换模型供应商DeepSeek、GLM、Kimi、MiniMax 等
MCP 管理管理 MCP 服务器配置
Skills / Prompts 管理同步和管理自定义技能与提示词
WebDAV 同步多设备间同步配置
内置 WebUI本地管理页面 http://127.0.0.1:15722/

CC Switch 的核心原理是管理环境变量配置——帮你把正确的配置写入系统，Claude Code 启动时直接读取，然后直连 DeepSeek。

4.2 安装 CC Switch

macOS

# Homebrew 安装（推荐）
brew install --cask cc-switch

或从 GitHub Releases 下载 .dmg 手动安装。首次打开时若提示"无法验证开发者"，前往系统设置 → 隐私与安全性，点击"仍要打开"。

Windows

从 GitHub Releases 下载 .msi 安装包，双击运行安装向导。

Linux

# Arch Linux (AUR)
paru -S cc-switch-bin

# 其他发行版：下载 .AppImage
chmod +x CC-Switch-*.AppImage
./CC-Switch-*.AppImage

4.3 添加 DeepSeek 提供商

  1. 点击左侧 "Providers"（提供商）菜单
  2. 点击"添加提供商"按钮
  3. 填写以下配置：
配置项填写内容说明
Provider NameDeepSeek自定义，随意填
Base URLhttps://api.deepseek.com/anthropic⚠️ 注意是 /anthropic 结尾
API Keysk-xxxxxxxxxxxxxxxxDeepSeek 平台创建的密钥
Opusdeepseek-v4-pro复杂任务主力模型
Sonnetdeepseek-v4-pro日常对话模型
Haikudeepseek-v4-flash简单高频任务轻量模型

4. 点击"保存"

💡 模型说明：
deepseek-v4-pro：DeepSeek V4 旗舰模型，推理能力最强
deepseek-v4-flash：轻量快速版，适合简单任务
将 Opus 和 Sonnet 指向同一模型是可行的——CC Switch 遵循 Claude Code 的三层模型映射机制。

4.4 选择 DeepSeek 为当前活跃提供商

  1. 在 CC Switch 主界面的提供商列表中，找到刚创建的 DeepSeek
  2. 点击右侧的"启用/切换"按钮
  3. 提示"已切换"，配置已生效

CC Switch 的设置是持久化的，不需要每次启动都重新配置。

4.5 关键注意事项

⚠️ 模型名不要带 [1m] 后缀：CC Switch 的本地路由模式下，模型名被原样转发。如果在模型名里写了 deepseek-v4-pro[1m]，会导致 DeepSeek 无法识别，可能回退到 Flash 模型。

路由模式 vs 代理模式：

模式原理适用场景
本地路由直接修改环境变量，直连 DeepSeek日常使用，推荐
代理模式启动本地代理服务转发请求需要日志/缓存时

日常使用推荐本地路由模式，性能最佳，无额外延迟。

五、验证一切正常

5.1 启动 Claude Code

cd 你的项目目录
claude

首次启动可能会询问初始设置问题（如是否同意数据收集等），如实回答即可。

5.2 简单对话测试

在 Claude Code 的交互提示符下输入：

你好，请用中文介绍一下你自己，并告诉我你是什么模型

如果看到类似以下回复，说明一切正常：

你好！我是 DeepSeek V4 Pro，由深度求索公司开发的大型语言模型。我目前正在通过 Claude Code 的框架为你服务……

5.3 如何确认当前走的是 DeepSeek

  • 方式一：查看用量 — 登录 platform.deepseek.com，进入"用量管理"，看 token 消耗量是否在增长
  • 方式二：查看 CC Switch 日志 — 在界面中查看最近请求记录
  • 方式三：开启调试日志 — export ANTHROPIC_LOG=debug 后启动 claude

六、进阶配置

6.1 VS Code 扩展版 Claude Code 的配置

  1. 在 VS Code 扩展市场搜索 "Claude Code" 并安装
  2. 确保 CC Switch 已配置并启用 DeepSeek（对 CLI 和扩展同时生效）
  3. 使用 Cmd+Shift+P（macOS）或 Ctrl+Shift+P（Windows），搜索 "Claude Code"
💡 提示：从终端启动 VS Code（code .）而非从 Dock/开始菜单启动，以确保环境变量正确传递。

6.2 多供应商切换

在 CC Switch 主界面，点击要切换的供应商旁的"启用"按钮即可，无需重启终端。

场景推荐供应商推荐模型
日常开发（主力）DeepSeekdeepseek-v4-pro
简单任务 / 高频调用DeepSeekdeepseek-v4-flash
特殊需求（免费额度）GLMglm-4-plus
超长上下文任务Kimimoonshot-v1-128k
追求最强推理（偶尔）Anthropic 官方claude-sonnet-4-6

6.3 MCP 服务器管理

CC Switch 内置了 MCP（Model Context Protocol）服务器管理功能。在 "MCP" 面板中，可以添加社区提供的 MCP 服务器、配置参数、一键启用/禁用，以及通过 WebDAV 同步配置到多台设备。

附录 A：备选方案

A.1 手动环境变量配置（不用 CC Switch）

直接在终端中设置环境变量，效果与 CC Switch 完全一致：

临时设置（仅当前终端会话）：

export ANTHROPIC_BASE_URL=https://api.deepseek.com/anthropic
export ANTHROPIC_AUTH_TOKEN=sk-你的DeepSeek-API-Key
export ANTHROPIC_MODEL=deepseek-v4-pro
export ANTHROPIC_DEFAULT_OPUS_MODEL=deepseek-v4-pro
export ANTHROPIC_DEFAULT_SONNET_MODEL=deepseek-v4-pro
export ANTHROPIC_DEFAULT_HAIKU_MODEL=deepseek-v4-flash
export CLAUDE_CODE_SUBAGENT_MODEL=deepseek-v4-flash

永久设置（推荐）：将上述 export 语句追加到 ~/.zshrc（或 ~/.bashrc），然后 source ~/.zshrc。

Windows (PowerShell 永久设置)：使用 [System.Environment]::SetEnvironmentVariable(...) 命令，设置后重启终端即可。

A.2 CLI 版切换工具

工具安装方式特点
cswitch-tuinpm install -g ccswitch-tuiTUI 交互界面，键盘操作
@bainiu/ccswitchnpx @bainiu/ccswitch零安装，直接运行

A.3 代理方案（高级 / 特殊场景）

对于企业统一 AI 网关、审计日志、请求缓存等场景，可考虑 LiteLLM 或 one-api 代理。个人开发者一般不需要——代理方案引入了额外的网络跳转和故障点。

附录 B：常见问题排查

B.1 npm 安装速度慢 / 失败

# 方案一：使用淘宝 npm 镜像
npm install -g @anthropic-ai/claude-code --registry=https://registry.npmmirror.com

# 方案二：永久切换 npm 源
npm config set registry https://registry.npmmirror.com

B.2 CC Switch 无法启动

macOS：提示"无法验证开发者"→ 系统设置 → 隐私与安全性 → 仍要打开。打开后闪退 → 尝试重新下载最新版。

Windows：杀毒软件拦截 → 添加为信任应用。缺少运行库 → 安装 Visual C++ Redistributable。

Linux：AppImage 无法运行 → 确保已安装 fuse：sudo apt install fuse。

B.3 对话报错 / 无响应

  1. 检查 API Key 是否完整（sk- 开头）
  2. 检查 Base URL 是 https://api.deepseek.com/anthropic（不是 /v1）
  3. 检查 DeepSeek 账户余额
  4. 用 curl 直连测试排除网络问题

B.4 模型能力感觉不对

  1. 确认没有回退到 flash 模型
  2. 确认模型名没有多余后缀（如 [1m]）
  3. 检查 CC Switch 当前启用的供应商
  4. 检查是否有环境变量与 CC Switch 冲突

B.5 DeepSeek 余额 / 充值

登录 platform.deepseek.com，在"用量管理"页面查看余额。支持微信支付 / 支付宝充值，建议设置余额告警避免突然欠费。

B.6 Git 相关问题

  • "git not found"：确认 Git 已安装且 PATH 配置正确；Windows 用户必须安装 Git for Windows
  • 非 Git 仓库报错：执行 git init && git add . && git commit -m "初始提交"

B.7 其他问题速查

问题可能原因解决方案
claude 命令找不到npm 全局路径未在 PATH检查 npm list -g --depth=0，确认安装路径
提示 "git version too old"Git 低于 2.23升级 Git 到 2.38+
CC Switch 保存后不生效需重启终端关闭当前终端窗口重新打开
VS Code 扩展找不到环境变量未传递从终端 code . 启动 VS Code
报错 "model not found"模型名拼写错误确认为 deepseek-v4-pro 或 deepseek-v4-flash
